
# 🧾 Technical Documentation — Bid MVP

## 📌 Overview

**Bid** is a web-based procurement platform that enables service requesters (clients) to create tenders and allows vendors to submit offers. This MVP version focuses on basic functionality for publishing public tenders, vendor registration, and offer submission.

---

## 🧱 Tech Stack

| Layer            | Technology              |
|------------------|--------------------------|
| Frontend         | Next.js + Tailwind CSS  |
| Backend          | Node.js (Express) or Ruby on Rails |
| Database         | PostgreSQL              |
| Hosting (Dev)    | Replit (for prototyping only) |
| Authentication   | Clerk/Auth.js (or Devise for Rails) |
| File Storage     | Local (Replit) or Cloudinary (optional) |

---

## 🧩 Core Features in the MVP

### 1. User Roles
- **Requester (Client)**: Can post tenders and view submitted offers.
- **Vendor (Supplier)**: Can browse public tenders and submit offers.

---

### 2. Tender Management
- `POST /tenders`: Create a new tender (title, description, deadline, attachments).
- `GET /tenders`: Retrieve list of public tenders.
- `GET /tenders/:id`: View tender details.

---

### 3. Offer Submission
- `POST /tenders/:id/offers`: Vendor submits an offer with:
  - Technical file (PDF)
  - Financial file (PDF)
  - Notes/comments

---

### 4. Authentication & Authorization
- `POST /auth/register`: Register as client or vendor.
- `POST /auth/login`: Login endpoint.
- `GET /me`: Returns user info and role.

---

### 5. Notifications (Basic)
- When a new offer is submitted, the requester is notified (via UI alert or email in full version).

---

## 🗃️ Database Schema (Simplified)

### `users`
```sql
id | name | email | password_hash | role (vendor/requester) | created_at
```

### `tenders`
```sql
id | title | description | deadline | requester_id (FK: users) | is_private | created_at
```

### `offers`
```sql
id | tender_id | vendor_id (FK: users) | technical_file_url | financial_file_url | notes | created_at
```

## 🚀 How to Run (Replit)

1. Fork the Replit project.
2. Add your environment variables:
   ```
   DATABASE_URL=
   JWT_SECRET=
   ```
3. Run the backend:
   ```bash
   cd backend
   npm install
   npm start
   ```
4. Run the frontend:
   ```bash
   cd frontend
   npm install
   npm run dev
   ```
