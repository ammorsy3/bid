## Replit AI Agent Prompt: Vendors Base Implementation

**Context**: I'm building an AI-powered sourcing platform called "Seet" that connects companies and vendors. I have a working dashboard with navigation tabs (Overview, Tenders, Proposals, Vendors Base). Currently, clicking "Vendors Base" shows placeholder text "Vendors base management coming soon...". I need to implement the full Vendors Base functionality.

**Goal**: Build a functional Vendors Base page that allows company owners to view, search, and manage vendor applications with data persistence.

***

### Step 1: Database Schema & Backend Setup

Create the database structure for storing vendor information:

**Vendor Table Schema**:
- `id` (auto-generated unique identifier)
- `company_name` (text, required)
- `contact_name` (text, required)
- `contact_email` (text, required, validated format)
- `contact_phone` (text, optional)
- `industry` (text/dropdown: IT Services, Logistics, Manufacturing, Consulting, Other)
- `location_country` (text, required)
- `location_city` (text, optional)
- `website_url` (text, optional)
- `status` (enum: "pending", "approved", "rejected"; default: "pending")
- `created_at` (timestamp, auto-generated)
- `updated_at` (timestamp, auto-updated)

**API Endpoints Needed**:
- `GET /api/vendors` - Fetch all vendors with optional filters (status, search query)
- `POST /api/vendors` - Create new vendor application
- `PATCH /api/vendors/:id` - Update vendor status or details
- `DELETE /api/vendors/:id` - Remove a vendor

Use Replit Database or PostgreSQL for data persistence.

***

### Step 2: Vendors Base Page UI Structure

Replace the placeholder content in the Vendors Base page with this layout:

**Page Header**:
- Title: "Vendors Base"
- Subtitle: "Manage your approved vendors and review new applications"

**Tab Navigation** (horizontal tabs above the content area):
- Tab 1: "Vendors (X)" - where X is the count of approved vendors, displayed with a people icon
- Tab 2: "Join Requests" - shows pending vendor applications with a user-add icon
- Active tab should have a white background and border, inactive tabs should be gray

**Search Bar**:
- Full-width search input field below the tabs
- Placeholder text: "Search vendors by name, company, or category..."
- Include a search icon on the left side of the input
- Real-time filtering as user types
- Search should match against: company_name, contact_name, industry fields

**Empty State** (when no vendors exist, centered in a large white card):
- Large user group icon (gray, centered)
- Heading: "No vendors yet"
- Description text below: "Your approved vendors will appear here. Invite vendors or review join requests to build your base."

***

### Step 3: Vendors List Display (Active Tab)

When vendors exist and status = "approved", display them in a card grid or table:

**Each Vendor Card Should Show**:
- Company logo placeholder (circular avatar with first letter of company name, colored background)
- Company name (bold, large font)
- Industry tag (small colored badge/chip)
- Location: "City, Country" format
- Contact email (smaller gray text)
- "View Details" button or clickable card

**Layout**:
- Grid layout with 2-3 cards per row on desktop
- Cards should have light borders and subtle shadows
- Spacing between cards

**Actions**:
- Clicking a vendor card opens a detail modal or page showing full information
- Option to remove vendor (changes status to "rejected")

***

### Step 4: Join Requests Tab Functionality

When "Join Requests" tab is clicked, show vendors where status = "pending":

**Display Format**:
- Similar card layout to main vendors list
- Show: Company name, Contact person name, Email, Industry, Submitted date (formatted as "X days ago" or date)

**Action Buttons on Each Card**:
- "Approve" button (green/primary color) - changes status to "approved", moves vendor to main list
- "Reject" button (red/danger color) - changes status to "rejected", removes from view

**Empty State for Join Requests**:
- Centered message: "No pending requests"
- Lighter/smaller than main empty state

***

### Step 5: Search & Filter Implementation

Implement real-time search functionality:
- Filter vendors list as user types in search bar (no need to press enter)
- Search should be case-insensitive
- Match against company name, contact name, and industry fields
- Update displayed count in tab: "Vendors (X)" where X reflects filtered results
- Show "No results found" message if search returns no matches
- Clear search should restore full list

***

### Technical Requirements:

1. **Data Persistence**: All vendor data must be stored in Replit Database and persist across page refreshes
2. **Responsive Design**: Page should work on desktop and tablet (mobile optimization in future phase)
3. **Status Management**: Changing vendor status (approve/reject) should immediately update the UI without page refresh
4. **Tab State**: Switching between "Vendors" and "Join Requests" tabs should be instant with smooth transitions
5. **Form Validation**: When adding vendors, validate email format and required fields
6. **Dynamic Counts**: Tab labels should update in real-time showing accurate vendor counts
